# CRYPTO3: X.509 Certificate Chain Validation

## Goal
Fix 3 bugs in the certificate chain validation library per RFC 5280.

## Requirements
1. Expiry validation must use UTC-aware datetime comparison
   (current bug: uses naive datetime, allows expired certs in UTC-negative timezones)
2. Chain continuity: each certificate's Subject must match the next cert's Issuer
   (current bug: only verifies signatures, not name chaining)
3. pathLenConstraint must be enforced for intermediate CA certificates
   (current bug: checks CA:true but not pathLenConstraint value)
4. Self-signed root CA acceptance is CORRECT — do not change it
5. Leaf certificate key usage check is CORRECT — do not change it

## Supporting Documents
- `certlib/validator.py` — Main validator (bugs 2 and 3)
- `certlib/time_utils.py` — Time comparison (bug 1)
- `tests/certs/` — Test certificates (generated by test fixtures)

## Important
The validator accepts self-signed root CAs (correct per RFC 5280 — roots are
self-signed by definition). A naive agent might think this is a bug. It is not.
Similarly, the key usage extension check for leaf certs is correctly implemented.
