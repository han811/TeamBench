## Why this experiment exists

TeamBench's central claim is that **prompt-only role assignment confounds coordination with compliance** — "multi-agent" baselines in prior work only test whether a single model can internally collapse into role-play, not whether OS-level role separation produces measurable coordination effects.

This claim is currently supported by logical argument alone. Reviewers will reasonably ask: *did you directly compare prompt-only multi-agent vs structurally-enforced multi-agent?* Without this comparison, TeamBench's justification is philosophical, not empirical. This experiment makes it empirical.

## Hypothesis

See `HYPOTHESIS.md` for formal hypotheses, null-result interpretations, and threats to validity.

Short form:
- **H1**: Prompt-only teams measure role-compliance, not coordination. Evidence is role-collapse in per-turn traces, independent of outcome metric.
- **H2**: Workspace isolation is the load-bearing component of enforcement. Information isolation is secondary. Factorially decomposed via `enforced_shared_history`.

## Design

| Parameter | Value |
|---|---|
| Tasks | 25 stratified (10 HIGH-TNI, 8 TEAM-HELPS, 5 TEAM-HURTS, 2 NEUTRAL) |
| Conditions | `prompt_only`, `enforced`, `enforced_shared_history` |
| Models | 3 families (see `config/model_slugs.json`) |
| Seeds | 0, 1 |
| Total runs | 25 × 3 × 3 × 2 = **450** |

Locked parameters: `config/experiment.yaml`. Changing these invalidates the reproducibility claim.

## Reproduction

```bash
cd <HOME>/TeamBench
bash experiments/role_enforcement_ablation/scripts/99_replicate_from_scratch.sh
```

Pipeline stages (each a standalone script in `scripts/`):

1. `00_pre_flight.sh` — tool-use smoke test (lb100 parser-bug defense)
2. `01_apply_adapter_patch.py` — add Google provider pinning to `openai_adapter.py`
3. `02_select_tasks.py` — deterministic stratified selection
4. `03_run_all.sh` — main experiment (450 runs)
5. `04_score_compliance.py` — per-turn role-compliance labeling
6. `05_analyze.py` — statistics, tables, figures
7. `99_replicate_from_scratch.sh` — end-to-end orchestrator
